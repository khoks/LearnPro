## Description

When the user clicks "Start session," they get a 3–5 item plan rendered in the sidebar (e.g., "Solve a list-comprehension problem", "Practice dict iteration", "One mini-debugging exercise"). The plan is generated by an LLM call seeded with: profile, last 5 episodes, current track position, time budget. As items complete, they tick off; if the user runs out of time mid-plan, unfinished items roll into tomorrow's plan.

**MVP scope:** session plan only. Daily/weekly/mastery plans are EPIC-006 v1+ work.

## Acceptance criteria

- [ ] Plan renders within 2s of session start (parallel with editor load).
- [ ] Plan is 3–5 items, each with a one-line objective and an estimated duration.
- [ ] Items tick off automatically when the corresponding problem is graded.
- [ ] Unfinished items at session end persist as `pending` for the next session.
- [ ] Plan generation logs an `agent_calls` row.
